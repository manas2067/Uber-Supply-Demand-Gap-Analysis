# Uber Supply Demand Gap Analysis

## Project Overview
This project analyzes the Uber Supply Demand Gap dataset to identify demand-supply mismatches, ride request trends, and customer service patterns using Python, SQL, and Microsoft Excel.

## Objectives
- Clean and preprocess the dataset.
- Perform Exploratory Data Analysis (EDA).
- Identify demand-supply gaps.
- Generate business insights.
- Build an interactive Excel dashboard.
- Perform SQL analysis on the dataset.

## Tools & Technologies
- Python
  - Pandas
  - NumPy
  - Matplotlib
  - Seaborn
- SQLite
- Microsoft Excel
- GitHub

## Project Workflow
1. Data Cleaning
2. Data Preprocessing
3. Exploratory Data Analysis (EDA)
4. Data Visualization
5. Business Insights
6. SQL Queries
7. Excel Dashboard

## Key Insights
- Trip Completed is the most common ride status.
- Demand is higher at City pickup points than Airport pickup points.
- Ride requests peak during morning and evening rush hours.
- No Cars Available increases significantly during peak demand periods.
- Supply-demand imbalance is most noticeable during rush hours.
